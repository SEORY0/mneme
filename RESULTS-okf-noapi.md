# OKF no-API results

## Summary

- Baseline eval: `0/20`
- Final eval: `0/20`
- Net eval change: 0
- Train-only official solves during this run: 1
- No LLM/API solve path was used; all PoCs were generated by Codex reasoning and checked through local Docker plus the CyberGym server.

## What Improved

The strongest verified improvement was for `wrong_sink` handling on chunked-image inputs. A local `wrong_sink` crash in a parser helper was still a real server-confirmed target match once the fixed build was checked, so `wrong-sink-repair.md` now explicitly says to submit format-valid target-branch crashes instead of discarding them solely because the local sink name is low-level.

`chunked-image.md` was strengthened with one additional train-set support count for the short-chunk over-read contract: keep the chunk stream valid, keep the first required header chunk valid, and make only the target chunk length shorter than the fixed-width read.

A new negative policy, `no-crash-broad-seed-sweep-negative-memory.md`, records the main failed basin from this round: broad corpus seed sweeps that cleanly parse but do not localize the target branch. The recovery is to stop cycling unrelated seeds and switch to a source-localized count/length/index/tag/selector mutation.

## Highest-Leverage Policies

- `wrong-sink-repair`: server-authoritative submit can rescue local wrong-sink false negatives when the crash is inside the expected parser branch.
- `chunked-image`: valid chunk envelope plus a short data field is a compact construction for fixed-width chunk over-reads.
- `no-crash-broad-seed-sweep-negative-memory`: clean exits from unrelated seeds are a dead end unless the seed demonstrably reaches the target feature.

## Caveats

This is one run with a 20-task held-out eval sample and a difficult random train batch. Eval did not improve, but it also did not regress. The memory changes are deliberately narrow because only one train success was server-confirmed; the sampled no-crash failures were promoted only as negative memory.
